# 6.7 Amazon FSx

## TL;DR
FSx = "RDS pero para filesystems" (managed third-party file systems)

## Los 4 tipos (memorizar para examen)

| FSx | Protocolo | Keyword en examen |
|-----|-----------|-------------------|
| Windows File Server | SMB | Windows, Active Directory, NTFS |
| Lustre | POSIX | HPC, Machine Learning, S3 integration |
| NetApp ONTAP | NFS/SMB/iSCSI | Migrar NAS on-prem, de-duplication |
| OpenZFS | NFS | Migrar ZFS, point-in-time cloning |

## Reglas rápidas

```
Windows + Active Directory    → FSx Windows
HPC + Machine Learning        → FSx Lustre
Migrar NAS + de-duplication   → FSx NetApp ONTAP
Migrar ZFS                    → FSx OpenZFS
```

## FSx Lustre extra
- Scratch = temporal, no replica, más rápido, más barato
- Persistent = replica en misma AZ, para datos importantes
- Se integra con S3 (lee/escribe directo)
