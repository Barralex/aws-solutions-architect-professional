# 5.19 Route 53 - Basics

## Record Types
| Tipo | Qué hace | Ejemplo |
|------|----------|---------|
| A | Hostname → IPv4 | example.com → 1.2.3.4 |
| AAAA | Hostname → IPv6 | example.com → 2001:db8::1 |
| CNAME | Hostname → otro hostname | www.example.com → example.com |
| Alias | Hostname → recurso AWS | example.com → ALB.amazonaws.com |
| NS | Name servers de la zona | Controla routing del dominio |

## CNAME vs Alias
| | CNAME | Alias |
|---|-------|-------|
| Target | Cualquier hostname | Solo recursos AWS |
| Root domain | ❌ NO funciona | ✅ SÍ funciona |
| Costo | Se cobra | Gratis |
| Health check | No nativo | Sí nativo |

**Alias targets:** ELB, CloudFront, API Gateway, S3 websites, Global Accelerator, Beanstalk
**NO puede ser Alias:** EC2 DNS name

## TTL (Time To Live)
- Tiempo que el cliente cachea la respuesta DNS
- Alto TTL (24h): menos tráfico DNS, cambios tardan en propagarse
- Bajo TTL (60s): más tráfico DNS, cambios rápidos
- Alias records: TTL automático (no se configura)

## Routing Policies

### Simple
- Un recurso, sin health check
- Puede retornar múltiples IPs (cliente elige random)

### Weighted
- % de tráfico a cada recurso
- Ej: 70% a EC2-1, 20% a EC2-2, 10% a EC2-3
- Con health checks

### Latency
- Redirige al recurso con menor latencia
- Basado en latencia usuario ↔ región AWS
- Con health checks

### Failover
- Primary + Secondary (DR)
- Health check en primary, failover automático

### Geolocation
- Rutea según ubicación del usuario (país, continente)
- Necesita record "default" para no matches

### Geoproximity
- Rutea por ubicación + bias
- Bias positivo (+1 a +99): atrae más tráfico
- Bias negativo (-1 a -99): repele tráfico
- Requiere Traffic Flow

### Multi-Value
- Retorna hasta 8 IPs healthy
- Con health checks
- NO reemplaza un load balancer

### IP-based
- Rutea según CIDR del cliente
- Define rangos de IP → endpoints específicos
