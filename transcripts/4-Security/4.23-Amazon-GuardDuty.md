# 4.23 Amazon GuardDuty

**Curso:** AWS Solutions Architect Professional (SAP-C02)
**Sección:** 4 - Security
**Duración:** ~4 min

---

## Resumen

GuardDuty = detección inteligente de amenazas con ML. Analiza CloudTrail, VPC Flow Logs, DNS logs + opcionales (S3, EKS, RDS, Lambda, EBS). Un click para activar, sin software. Detecta cryptocurrency attacks.

---

## Inputs de GuardDuty

| Siempre activos | Opcionales |
|-----------------|------------|
| CloudTrail Events (Management + Data) | S3 Data Events |
| VPC Flow Logs | EKS Audit Logs |
| DNS Logs | RDS/Aurora Login Events |
| | Lambda Network Activity |
| | EBS Volumes |

---

## Flujo

```
CloudTrail ──┐
VPC Flow  ───┼──► GUARDDUTY (ML) ──► Findings ──► EventBridge
DNS Logs  ───┘                                        │
+ opcionales                                          ▼
                                              Lambda, SNS, etc.
```

---

## Delegated Administrator

En Organizations, una cuenta miembro puede ser **Delegated Administrator** de GuardDuty:
- Puede habilitar/administrar GuardDuty en TODAS las cuentas
- Solo la Management Account puede designarlo

---

## Puntos Clave

1. **ML + anomaly detection** - sin software
2. **Cryptocurrency attacks** - tiene finding dedicado
3. **Delegated Administrator** - para Organizations
4. **EventBridge** - para automatizar respuestas
