# 6.9 AWS DataSync

## Qué es
Mover grandes cantidades de datos de un lugar a otro.

## Fuentes y Destinos

```
DESDE                          HACIA
─────                          ─────
On-premises (NFS/SMB/HDFS)  →  S3 (cualquier clase, incluso Glacier)
Otro cloud                  →  EFS
AWS service                 →  FSx (todos)

(Bidireccional - también de AWS a on-prem)
```

## Agent

```
On-prem / Otro cloud → AWS     = NECESITA AGENT
AWS → AWS                      = NO necesita agent
```

## Características clave

| Feature | Detalle |
|---------|---------|
| Scheduling | Hourly, daily, weekly (NO es continuo) |
| Metadata | Preserva permisos y metadata (NFS POSIX, SMB) |
| Performance | Hasta 10 Gbps por tarea |
| Bandwidth | Podés limitar para no saturar red |

## Snowcone + DataSync

Si no tenés red suficiente:
```
Snowcone tiene DataSync agent PRE-INSTALADO

On-prem → Snowcone (copia data) → Ship a AWS → Sync a S3/EFS/FSx
```

## AWS to AWS Sync

```
S3  ◄──────►  EFS  ◄──────►  FSx

Sin agent, preserva metadata
```

## Para el examen

```
Mover data on-prem → AWS        → DataSync + Agent
Mover data AWS → AWS            → DataSync (sin agent)
Preservar permisos/metadata     → DataSync (único que lo hace)
Sin red suficiente              → Snowcone (tiene DataSync pre-installed)
Sync continuo                   → NO es DataSync (es scheduled)
```
