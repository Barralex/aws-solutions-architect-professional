## Big Data Ingestion Pipeline

```
IoT Devices
    │
    ▼
Kinesis Data Streams (real-time)
    │
    ▼
Kinesis Data Firehose (near real-time, ~1 min buffer)
    │
    ├── Lambda (transformar data)
    │
    ▼
S3 Ingestion Bucket
    │
    ├── S3 Event → SQS (opcional) → Lambda
    │   o
    ├── S3 Event → Lambda directo
    │
    ▼
Athena (query S3)
    │
    ▼
S3 Reporting Bucket
    │
    ├── QuickSight (dashboards)
    └── Redshift (warehouse + queries)
```

## Comparación Warehousing

| Criterio | EMR | Athena | Redshift |
|----------|-----|--------|----------|
| **Cuándo** | Migrar Big Data on-prem | Queries simples/esporádicas | SQL avanzado, uso sostenido |
| **Herramientas** | Hadoop, Spark, Hive | SQL serverless | SQL + Spectrum |
| **Data source** | S3 (EMRFS), DynamoDB (Hive), EBS (HDFS) | Solo S3 | S3 (Spectrum) o cargada en cluster |
| **Infra** | Clusters (long-running o per-job) | Serverless | Provisioned (o Serverless) |
| **Pricing** | Spot/On-Demand/Reserved por nodo | $5/TB escaneado | Por nodo, uso sostenido |
| **Storage** | HDFS (scratch, EBS) o EMRFS (S3 long-term) | S3 | Columnar interno + S3 |

## Reglas rápidas para el examen

| Ves en pregunta | Respuesta |
|-----------------|-----------|
| Migrar Hadoop/Spark a AWS | EMR |
| Queries esporádicas SQL sobre S3 | Athena |
| Data warehouse con SQL avanzado | Redshift |
| Query S3 sin cargar datos (con Redshift) | Redshift Spectrum |
| Pipeline IoT → analytics | Kinesis Streams → Firehose → S3 → Athena/Redshift |
| Dashboards sobre cualquier fuente | QuickSight |
| Cluster largo con muchos jobs | EMR long-running + auto-scaling |
| Cluster por job independiente | EMR transient clusters |
