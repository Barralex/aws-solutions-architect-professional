# 5.1 Solution Architecture on AWS

**Curso:** AWS Solutions Architect Professional (SAP-C02)
**Sección:** 5 - Compute & Load Balancing
**Duración:** ~4 min

---

## Resumen

Visión general de las capas de una arquitectura AWS típica: DNS → CDN → Web Layer → Compute → Database → Caching → Storage → Decoupling. Cada elección impacta costo, disponibilidad, escalabilidad.

---

## Servicios por capa

| Capa | Servicios |
|------|-----------|
| **DNS** | Route 53 |
| **CDN** | CloudFront |
| **Static Assets** | S3, Glacier |
| **Web Layer** | CLB, ALB, NLB, API Gateway, Elastic IP |
| **Compute** | EC2, ASG, Lambda, ECS, Fargate, Batch, EMR |
| **Database** | RDS, Aurora, DynamoDB, ElasticSearch, S3, Redshift |
| **Caching/Session** | ElastiCache, DAX, DynamoDB |
| **Storage** | EBS, EFS, Instance Store |
| **Decoupling** | SQS, SNS, Kinesis, Amazon MQ, Step Functions |

---

## Punto clave

Cada elección impacta:
- Costo
- Disponibilidad
- Durabilidad
- Escalabilidad

El examen te pregunta: **¿Cuál es el mejor servicio para este escenario?**
