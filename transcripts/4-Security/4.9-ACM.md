# 4.9 AWS Certificate Manager (ACM)

**Curso:** AWS Solutions Architect Professional (SAP-C02)
**Sección:** 4 - Security
**Duración:** 4 min

---

## Resumen

ACM provisiona y renueva SSL certificates gratis. Se integra con ALB, CloudFront, API Gateway. Es REGIONAL (no se pueden copiar certs entre regiones).

---

## Lo que hace ACM

```
┌─────────────────────────────────────────────────────────────┐
│  DOS OPCIONES:                                              │
│                                                             │
│  1. Subir tu propio cert (manual renewal)                  │
│  2. ACM lo provisiona (auto renewal, GRATIS)               │
└─────────────────────────────────────────────────────────────┘
```

## Integraciones

```
┌─────────────────────────────────────────────────────────────┐
│  ACM se integra con:                                        │
│                                                             │
│  • ALB / NLB                                                │
│  • CloudFront                                               │
│  • API Gateway                                              │
│  • Elastic Beanstalk (load balancers)                      │
└─────────────────────────────────────────────────────────────┘
```

## SSL Termination

```
┌─────────────────────────────────────────────────────────────┐
│  User ──HTTPS──► ALB ──HTTP──► EC2                         │
│                   │                                         │
│                   └── SSL termination aquí                 │
│                       (EC2 no gasta CPU en SSL)            │
│                                                             │
│  ACM mantiene el cert en el ALB automáticamente            │
└─────────────────────────────────────────────────────────────┘
```

## Public vs Private Certs

```
┌─────────────────────────────────────────────────────────────┐
│  PUBLIC CERT                                                │
│  • Para apps públicas                                      │
│  • Debe ser de CA conocida                                 │
│  • Verificar DNS público                                   │
│                                                             │
│  PRIVATE CERT                                               │
│  • Para apps internas                                      │
│  • Tu propio Private CA                                    │
│  • Apps deben confiar en tu CA                             │
└─────────────────────────────────────────────────────────────┘
```

## ACM es REGIONAL

```
┌─────────────────────────────────────────────────────────────┐
│  ⚠️  MUY IMPORTANTE PARA EL EXAMEN                         │
│                                                             │
│  ACM = REGIONAL                                             │
│  NO se pueden copiar certs entre regiones                  │
│                                                             │
│  Si tenés ALBs en 3 regiones:                              │
│  → Necesitás 3 certificados (uno por región)               │
│                                                             │
│  CloudFront es global → un solo cert                       │
└─────────────────────────────────────────────────────────────┘
```

---

## Puntos Clave para el Examen

1. **ACM provisiona y renueva gratis** (si lo genera ACM)
2. **Cert subido manualmente** = renewal manual
3. **ACM es REGIONAL** = un cert por región
4. **No se copian certs** entre regiones
5. **CloudFront** = global, un solo cert
6. **ALB multi-region** = un cert ACM por región
