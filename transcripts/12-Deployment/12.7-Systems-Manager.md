# 12.7 AWS Systems Manager (SSM)

## Conceptos Clave

### Qué es SSM
- Gestionar **EC2 instances y servidores on-premises a escala**
- Servicio **gratuito**
- Funciona con Linux y Windows
- Integrado con CloudWatch, AWS Config, CloudTrail

### SSM Agent
- Necesita estar instalado en cada máquina
- **Preinstalado** en Amazon Linux AMI y algunos Ubuntu AMI
- Si la instancia no aparece en SSM:
  1. Agent mal configurado
  2. Falta **IAM Role** (EC2) o **access keys** (on-premises)

### Run Command
- Ejecutar scripts/comandos en **múltiples instancias sin SSH**
- No necesita puerto 22 abierto
- **Rate control** (cuántas a la vez) + **Error control** (parar o seguir)
- Integrado con IAM y CloudTrail

#### Arquitectura: Run Command + ASG Lifecycle Hook
```
ASG termina instancia → Lifecycle Hook (Terminating:Wait)
  → EventBridge (rule) → SSM Run Command (extraer logs)
  → Completar lifecycle → ASG termina la instancia
```

### Patch Manager
- Parchear instancias a escala (Linux + Windows)
- **Patch Baseline** → qué parches aplicar
- **Patch Groups** → a quién (Dev/Test/Prod)
- **Maintenance Window** → cuándo
- **RunPatchBaseline** → ejecutar
- **SSM Inventory** → verificar compliance

### Session Manager (CLAVE PARA EXAMEN)
- Shell segura **sin SSH, sin puerto 22, sin bastion host, sin SSH keys**
- Conectar via consola, CLI o SDK
- Soporta Linux, macOS, Windows
- **Ventaja sobre SSH:** auditoría completa
  - Comandos logueados a **CloudWatch Logs** y **S3**
  - Sesiones auditadas via **CloudTrail** (StartSession events)

### OpsCenter
- Centro de operaciones para resolver problemas (OpsItems)
- Agrega info de: Config, CloudTrail, CloudWatch, CloudFormation
- Se crean OpsItems via CloudWatch Alarms o EventBridge
- Resolver con **SSM Automation Runbooks**

## Reglas para el examen
- "Ejecutar comandos sin SSH" → **Run Command**
- "Shell sin puerto 22 + auditoría" → **Session Manager**
- "Parchear a escala" → **Patch Manager**
- "Instancia no aparece en SSM" → **Agent o IAM Role**
- "Un solo lugar para resolver issues operativos" → **OpsCenter**
