# 5.3 High Performance Computing (HPC)

**Curso:** AWS Solutions Architect Professional (SAP-C02)
**Sección:** 5 - Compute & Load Balancing
**Duración:** ~7 min

---

## Resumen

HPC en AWS = muchas instancias haciendo cálculos pesados, pagas solo lo que usas, destruyes cuando terminas. Servicios clave: Direct Connect/Snowball (data transfer), EC2 Cluster placement + ENA/EFA (networking), FSx for Lustre (storage), AWS Batch/ParallelCluster (orchestration).

---

## ¿Por qué HPC en la nube?

- Crear miles de recursos al instante
- Pagar solo lo usado
- Destruir todo al terminar
- Casos: genomics, ML, weather prediction, autonomous driving

---

## Servicios por categoría

### 1. Data Transfer (mover datos a AWS)

| Servicio | Uso |
|----------|-----|
| **Direct Connect** | Gbps constante, red privada |
| **Snowball** | Petabytes físicamente |
| **DataSync** | On-prem NFS/SMB → S3/EFS/FSx |

### 2. Compute & Networking

| Servicio | Uso |
|----------|-----|
| **EC2** | CPU/GPU optimized |
| **Spot Instances** | Ahorro masivo |
| **Cluster Placement Group** | 10 Gbps, baja latencia |

### 3. Enhanced Networking

```
┌─────────────────────────────────────────────────┐
│  ENI (Elastic Network Interface)               │
│  → Básico, virtual NIC                         │
├─────────────────────────────────────────────────┤
│  ENA (Elastic Network Adapter)                 │
│  → Hasta 100 Gbps                              │
│  → Higher bandwidth, lower latency             │
├─────────────────────────────────────────────────┤
│  EFA (Elastic Fabric Adapter)                  │
│  → Solo Linux                                  │
│  → Bypasea el OS para aún menos latencia       │
│  → Para HPC distribuido (MPI)                  │
│  → Tightly coupled workloads                   │
└─────────────────────────────────────────────────┘
```

### 4. Storage

| Servicio | IOPS | Notas |
|----------|------|-------|
| **EBS io2 Block Express** | 256,000 | Attached storage |
| **Instance Store** | Millones | Se pierde si EC2 muere |
| **S3** | - | Blobs grandes, no file system |
| **EFS** | Escala con tamaño | Provisioned IOPS disponible |
| **FSx for Lustre** | Millones | HPC optimized, backed by S3 |

### 5. Orchestration

| Servicio | Uso |
|----------|-----|
| **AWS Batch** | Jobs paralelos multi-nodo |
| **ParallelCluster** | Open source, crea VPC/subnets/clusters |

---

## Puntos Clave Examen

1. **ENA** = 100 Gbps, enhanced networking
2. **EFA** = solo Linux, bypasea OS, para MPI/HPC distribuido
3. **FSx for Lustre** = storage HPC optimizado
4. **Cluster placement** = baja latencia entre instancias
5. **AWS Batch** = orchestration de jobs HPC
