# 10.5 Amazon MSK (Managed Streaming for Apache Kafka)

## Qué es

- **Kafka managed en AWS**
- Alternativa a Kinesis Data Streams
- Cluster fully managed (brokers + ZooKeeper)

## Deployment

- En tu VPC
- Multi-AZ (hasta 3) para HA
- Auto-recovery de fallos
- Storage en EBS (retención ilimitada)

## MSK vs MSK Serverless

| | MSK | MSK Serverless |
|---|-----|----------------|
| Provisioning | Vos elegís brokers | Auto |
| Scaling | Manual | Auto |
| Pricing | Por broker/hora | Por uso |

## Kinesis vs MSK

| | Kinesis Data Streams | Amazon MSK |
|---|---------------------|------------|
| Unidad | Shard | Partition |
| Max mensaje | 1 MB | 1 MB (configurable hasta 10MB) |
| Retención | 1-365 días | Ilimitada (EBS) |
| Escalar UP | Shard splitting | Add partitions |
| Escalar DOWN | Shard merging | NO se puede |
| Encryption | TLS | Plaintext o TLS |
| Protocolo | AWS SDK | Kafka protocol |

## Producers

- Kafka Producer (cualquier librería Kafka)

## Consumers

- Kinesis Data Analytics (Apache Flink)
- AWS Glue (Spark Streaming)
- Lambda (event source)
- Custom consumer (EC2, ECS, EKS)

## Para el examen

| Ves en pregunta | Respuesta |
|-----------------|-----------|
| Kafka managed en AWS | Amazon MSK |
| Migrar Kafka on-prem a AWS | Amazon MSK |
| Streaming con protocolo Kafka | MSK |
| Retención > 1 año | MSK (ilimitada) |
| Mensaje > 1MB | MSK (hasta 10MB) |
| Reducir particiones | NO se puede en MSK |
