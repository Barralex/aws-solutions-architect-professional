# 12.6 AWS CDK - Cloud Development Kit

## Conceptos Clave

### Qué es CDK
- Framework para definir infraestructura cloud usando **lenguajes de programación**
- Soporta: JavaScript/TypeScript, Python, Java, .NET
- El código se **compila a CloudFormation templates** (JSON/YAML)

### Por qué usar CDK
- **Type safety** - el IDE detecta errores antes de deployar
- **Reutilización** - funciones, clases, for loops, condicionales
- **Mismo lenguaje** para infra y app (ej: Lambda en Python + CDK en Python)
- Ideal para Lambda functions y Docker containers (ECS/EKS)

### Flujo de CDK

```
CDK App (Python/TS/Java)
        │
        │  cdk synth
        ▼
CloudFormation Template (JSON/YAML)
        │
        │  cdk deploy
        ▼
CloudFormation (deploy real)
```

### CDK vs SAM vs CloudFormation (CLAVE PARA EXAMEN)

| Herramienta | Lenguaje | Por debajo | Cuándo |
|---|---|---|---|
| CloudFormation | YAML/JSON | Es la base | Infra general |
| SAM | YAML | CF + CodeDeploy | Serverless (Lambda traffic shifting) |
| CDK | Python, TS, Java, .NET | Compila a CF | Programar infra con type safety |

### Regla para el examen
- "Programming language" / "type safety" / "familiar language" → **CDK**
- "Serverless" / "Lambda traffic shifting" / "YAML serverless" → **SAM**
- "YAML/JSON template" directo → **CloudFormation**
