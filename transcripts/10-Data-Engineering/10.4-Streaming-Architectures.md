# 10.4 Streaming Architectures & Comparisons

## Arquitectura típica real-time

```
Producers → Kinesis Data Streams → EC2 / Flink / Firehose → S3 / Redshift / OpenSearch
```

También se puede producir directo a Firehose.

## Comparación de costos: Kinesis vs DynamoDB Streams

### Escenario: 3000 mensajes de 1KB/segundo

| Opción | Costo mensual |
|--------|---------------|
| Kinesis Streams (3 shards) | ~$32/mes |
| DynamoDB + Streams (3000 WCU) | ~$1,450/mes |

**Factor 50x más caro con DynamoDB para streaming**

Kinesis + Firehose → S3 para storage largo plazo

## Tabla comparativa de tecnologías

| Servicio | Data | Retención | Orden | Scale | Readers | Latencia |
|----------|------|-----------|-------|-------|---------|----------|
| **Kinesis Streams** | Inmutable | 1-365 días | Por shard | Provision shards | EC2, Lambda, Firehose, Flink, KCL | 200ms |
| **Firehose** | Inmutable | Sin storage | - | Auto | Destinations only | ~1 min |
| **SQS Standard** | Inmutable | 1-14 días | No | Ilimitado | EC2, Lambda | 10-100ms |
| **SQS FIFO** | Inmutable | 1-14 días | Por Group ID | 300/s (3K batch) | EC2, Lambda | 10-100ms |
| **SNS** | Inmutable | Sin retención | No | Ilimitado | HTTP, Lambda, Email, SQS | 10-100ms |
| **DynamoDB** | Mutable | Infinito/TTL | No | WCU/RCU + Auto | SDK, Streams→Lambda | 10-100ms |
| **S3** | Mutable* | Infinito/Lifecycle | No | 3500 PUT/s/prefix | SDK, S3 Events | 10-100ms |

*S3 tiene versionado para inmutabilidad opcional

## Para el examen

| Ves en pregunta | Respuesta |
|-----------------|-----------|
| Streaming más cost-effective | Kinesis (NO DynamoDB Streams) |
| Real-time + storage largo plazo | Kinesis Streams + Firehose → S3 |
| Ordenar mensajes | Kinesis (por shard) o SQS FIFO (por group) |
| Máxima escala sin orden | SQS Standard |
| Datos mutables | DynamoDB o S3 |
| Latencia más baja | SQS, SNS, DynamoDB (10-100ms) |
