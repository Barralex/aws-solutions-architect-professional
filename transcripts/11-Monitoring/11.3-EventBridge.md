# 11.3 - Amazon EventBridge

## Transcript
Amazon EventBridge (formerly CloudWatch Events): event buses, sources, destinations, archive/replay, schema registry, resource-based policies.

## Key Concepts

### EventBridge = formerly CloudWatch Events
- React to events or schedule cron jobs in the cloud

### Event Sources
- **AWS Services**: EC2 (start/stop/terminate), CodeBuild (build fail), S3 (upload), Trusted Advisor (findings)
- **CloudTrail integration**: intercept ANY API call
- **Schedule/Cron**: every 4 hours, every Monday at 8am, first Monday of month, etc.

### Event Destinations
- Lambda, AWS Batch, ECS Task
- SQS, SNS, Kinesis Data Streams
- Step Functions, CodePipeline, CodeBuild
- SSM Automation, EC2 Actions (start/stop/restart)

### Three Types of Event Buses
1. **Default Event Bus**: AWS services events
2. **Partner Event Bus**: SaaS partners (Zendesk, Datadog, Auth0, etc.)
3. **Custom Event Bus**: your own applications' events

### Event Archive & Replay
- Archive all events or filter a subset
- Retention: indefinite or set period
- **Replay**: re-send archived events (debugging, fix production, retest)

### Schema Registry
- EventBridge analyzes events and infers schema
- Generate code for your application from the schema
- Schemas can be versioned

### Resource-Based Policies
- Manage permissions for event buses
- Allow/deny events from other regions or accounts
- Use case: **central event bus** for AWS Organization
- Other accounts do `PutEvents` to central bus

### Cross-Account Pattern
- Central event bus in security/logging account
- Resource-based policy allows other accounts to send events
- Aggregates events from entire organization
