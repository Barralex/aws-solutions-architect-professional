# 12.3 - CloudFormation

## Resumen
IaC nativo de AWS. Backbone de Beanstalk, Service Catalog y SAM. Para SA Pro hay que conocer algunas features específicas.

## Conceptos Clave

### Deletion Policies
- `Retain` → no borra el recurso al eliminar la stack
- `Snapshot` → toma snapshot antes de borrar (EBS, ElastiCache Redis, RDS, Redshift)
- `Delete` → default, borra todo
- Excepción: RDS DBCluster default es Snapshot
- S3 bucket debe estar vacío para poder borrarse

### Custom Resources (Lambda)
- Para recursos no soportados por CloudFormation
- Para recursos on-premise
- Para lógica custom (vaciar S3 bucket, obtener AMI ID)
- Backed by Lambda function
- Lambda se invoca en create/update/delete de la stack

### StackSets
- Desplegar la misma stack en múltiples cuentas y regiones
- Admin account → crea StackSets
- Target accounts → reciben las stack instances
- Auto-deployment: con AWS Organizations, deploya automáticamente en cuentas nuevas

### Drift Detection
- Detecta cambios manuales hechos fuera de CloudFormation
- Compara estado actual vs estado definido en la template
- Puede evaluar stack completa o recursos individuales

### Secrets Manager Integration
- Generar secreto en template
- Referenciarlo en RDS instance
- SecretTargetAttachment vincula secreto a RDS → rotación automática

### Resource Import
- Importar recursos existentes (creados fuera de CF) a una stack
- No recrea el recurso, solo lo adopta
- Requiere: template con el recurso + identificador único
- Cada recurso importado necesita DeletionPolicy definida
- No se puede importar mismo recurso en múltiples stacks
