# 4.14 S3 Multi-Region Access Points

**Curso:** AWS Solutions Architect Professional (SAP-C02)
**Sección:** 4 - Security
**Duración:** 5 min

---

## Resumen

Multi-Region Access Points proveen un endpoint global que rutea automáticamente al bucket S3 más cercano (menor latencia). Requiere replicación bidireccional entre buckets y soporta failover activo/pasivo.

---

## Por qué existe

```
┌─────────────────────────────────────────────────────────────┐
│  PROBLEMA: App global, usuarios en todo el mundo            │
│                                                             │
│  Usuario Tokyo ──────────────► s3://bucket-us-east-1       │
│                    (latencia alta, 200ms+)                  │
│                                                             │
│  SOLUCIÓN: Multi-Region Access Point                        │
│                                                             │
│  Usuario Tokyo ──► MRAP ──► s3://bucket-ap-northeast-1     │
│                    (ruteo automático, ~20ms)                │
└─────────────────────────────────────────────────────────────┘
```

---

## Arquitectura

```
┌─────────────────────────────────────────────────────────────┐
│                                                             │
│            ┌──────────────────────────┐                     │
│            │  Multi-Region Access     │                     │
│            │  Point (MRAP)            │                     │
│            │  xxxxxx.mrap.accesspoint │                     │
│            │  .s3-global.amazonaws.com│                     │
│            └────────────┬─────────────┘                     │
│                         │                                   │
│         ┌───────────────┼───────────────┐                   │
│         ▼               ▼               ▼                   │
│    ┌─────────┐    ┌─────────┐    ┌─────────┐               │
│    │ Bucket  │◄──►│ Bucket  │◄──►│ Bucket  │               │
│    │us-east-1│    │eu-west-1│    │ap-north │               │
│    └─────────┘    └─────────┘    └─────────┘               │
│                                                             │
│    Replicación BIDIRECCIONAL (S3 Cross-Region Replication) │
└─────────────────────────────────────────────────────────────┘
```

---

## Failover Controls

```
┌─────────────────────────────────────────────────────────────┐
│  ACTIVE-ACTIVE (default):                                   │
│  • Todos los buckets reciben tráfico                       │
│  • Ruteo por latencia                                      │
│                                                             │
│  ACTIVE-PASSIVE:                                            │
│  • Un bucket "activo" recibe todo                          │
│  • Otros "pasivos" solo para failover                      │
│  • Cambio manual o automático                              │
│                                                             │
│  Casos de uso:                                              │
│  • DR: región primaria falla → failover a secundaria       │
│  • Compliance: forzar región específica temporalmente      │
└─────────────────────────────────────────────────────────────┘
```

---

## Quién lo usa

- Empresas globales con usuarios en múltiples continentes
- Gaming (baja latencia crítica)
- Streaming de contenido global
- Aplicaciones que necesitan DR automático

---

## Puntos Clave para el Examen

1. **MRAP** = UN endpoint global, múltiples buckets
2. **Ruteo automático** por menor latencia
3. **Requiere S3 Replication** bidireccional entre buckets
4. **Failover controls** = activo/pasivo o activo/activo
5. **"Global users, low latency S3"** = Multi-Region Access Points
