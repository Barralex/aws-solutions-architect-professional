# 12.1 - Elastic Beanstalk

## Resumen
Elastic Beanstalk es una plataforma PaaS (Platform as a Service) que envuelve componentes AWS conocidos (EC2, ASG, ELB, RDS) en una vista centrada en el desarrollador. Es ideal para re-platforming de aplicaciones on-premise a AWS.

## Conceptos Clave

### Qué es Beanstalk
- Vista centrada en el desarrollador para desplegar aplicaciones en AWS
- Wrapper alrededor de EC2, ASG, Elastic IP, ELB, RDS, etc.
- Control total sobre la configuración de cada componente
- Servicio gratuito (pagas por las instancias subyacentes)
- Servicio gestionado: configuración de instancia y OS manejada por Beanstalk
- Tu responsabilidad: solo el código de la aplicación

### Plataformas Soportadas
- Go, Java, Java con Tomcat, .NET (Windows Server)
- Node.js, PHP, Python, Ruby, Packer
- Docker: Single Container, Multi-Container, Preconfigured
- Si puedes dockerizar tu app → puedes migrarla a Beanstalk

### Modelos de Arquitectura
1. **Single Instance** → Dev
   - Una AZ, una EC2, Elastic IP
   - Opcionalmente RDS single-AZ
   - Rentable para desarrollo

2. **Load Balancer + ASG** → Producción web
   - Multi-AZ, ALB + Auto Scaling Group
   - RDS Multi-AZ (master + standby)
   - Alta disponibilidad

3. **ASG Only (Worker Tier)** → Producción no-web
   - Cola SQS gestionada por Beanstalk
   - EC2 ASG que lee de la cola
   - Escala según la carga de la cola

### Web Tier vs Worker Tier
- Tareas largas (procesar video, generar ZIP, filtros de imágenes) → Worker Tier
- Desacoplar = SQS (clave para el examen)
- Worker Tier puede tener tareas periódicas via `cron.yaml`
- Arquitectura típica: Web Tier (ALB + ASG) → SQS → Worker Tier (ASG)

### Blue/Green Deployment
- NO es una feature directa de Beanstalk
- Crear nuevo entorno "stage" con v2
- Route 53 weighted routing para mover tráfico gradualmente (90/10 → 80/20 → ...)
- Beanstalk "Swap URL" = intercambio de DNS (cambia todo el tráfico de golpe)
- Zero downtime, fácil rollback (borrar el entorno verde)

## Para el Examen
- Beanstalk = re-platforming de on-premise a la nube
- Tres modelos de arquitectura: Single Instance, LB+ASG, Worker Tier
- Desacoplar tareas largas → Worker Tier con SQS
- Blue/Green no es feature nativa → se logra con Route 53 weighted + Swap URL
- Java con Tomcat on-premise → Beanstalk es la respuesta
