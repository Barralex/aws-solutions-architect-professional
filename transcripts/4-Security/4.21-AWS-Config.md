# 4.21 AWS Config

**Curso:** AWS Solutions Architect Professional (SAP-C02)
**Sección:** 4 - Security
**Duración:** ~5 min

---

## Resumen

Config = audita y registra compliance de recursos AWS. NO PREVIENE acciones, solo detecta y notifica. Tiene 75+ managed rules, puedes crear custom con Lambda. Puede remediar automáticamente con SSM Automation.

---

## Config NO es preventivo

```
SCP         → PREVIENE acciones (deny)
AWS Config  → DETECTA non-compliance (audit)
```

---

## Qué hace Config

| Feature | Descripción |
|---------|-------------|
| **Registra cambios** | Configuración de recursos en el tiempo |
| **Evalúa compliance** | Reglas que marcan compliant/non-compliant |
| **Alertas** | SNS cuando algo cambia |
| **Historial** | Ver quién cambió qué (con CloudTrail) |
| **Agregación** | Multi-cuenta/región a cuenta central |

---

## Reglas

- **75+ managed rules** de AWS
- **Custom rules** con Lambda
- **Triggers:** por cambio de config O intervalo de tiempo

---

## Remediación automática

```
Recurso non-compliant
        │
        ├──► EventBridge ──► Lambda, SNS, etc.
        │
        └──► SSM Automation ──► Remediar automáticamente
                               (ej: cerrar SSH en SG)
```

---

## Puntos Clave

1. **NO previene** - solo detecta y audita
2. **Por región** - habilitar en cada región
3. **Remediación** - SSM Automation
4. **Agregación** - multi-cuenta a cuenta central
