# 10.1 Kinesis Data Streams

## Qué es

- **Kafka serverless de AWS**
- Collect + store streaming data en real-time
- Keyword examen: **real-time**

## Arquitectura

```
Producers → Kinesis Data Streams → Consumers
(apps, agents)    (shards)         (Lambda, Firehose, Flink, apps)
```

## Conceptos clave

| Kinesis | Kafka equivalente |
|---------|-------------------|
| Stream | Topic |
| Shard | Partition |
| Partition Key | Partition Key |
| Data retention | Log retention |

## Features

- Retención: 1-365 días
- Replay de data (inmutable, no se puede borrar)
- Max message: 10MB
- Orden garantizado por partition key
- Encryption: KMS at-rest, HTTPS in-flight

## Capacity Modes

### Provisioned Mode
- Elegís # de shards manualmente
- 1 shard = 1 MB/s IN, 2 MB/s OUT, 1000 records/s
- Escalar manualmente
- Pago: por shard/hora

### On-Demand Mode
- Sin provisioning
- Default: 4 MB/s IN (4000 records/s)
- Auto-scale basado en últimos 30 días
- Pago: por stream/hora + data in/out

## Libraries

- **KPL** (Kinesis Producer Library): para alto throughput
- **KCL** (Kinesis Client Library): para consumers optimizados

## Para el examen

| Ves en pregunta | Respuesta |
|-----------------|-----------|
| Real-time streaming | Kinesis Data Streams |
| Replay/reprocess data | Kinesis (retención hasta 365 días) |
