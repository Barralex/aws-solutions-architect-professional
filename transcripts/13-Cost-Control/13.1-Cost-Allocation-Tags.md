# 13.1 Cost Allocation Tags

## Conceptos Clave

### Qué son
- Tags normales de AWS que aparecen como **columnas en reportes de billing**
- Permiten desglosar costos por proyecto, equipo, ambiente, etc.

### Dos tipos de tags

| Tipo | Prefijo | Quién las crea | Retroactiva? |
|---|---|---|---|
| AWS Generated | `aws:` | AWS automáticamente | NO (solo recursos nuevos) |
| User Tags | `user:` | Vos | Se pueden agregar a cualquier recurso |

### Datos clave para el examen
- Solo aparecen en la **consola de Billing** (no en EC2, S3, etc.)
- Tardan **~24 horas** en aparecer en reportes
- Hay que **activarlas** en la consola de Billing
- AWS Generated Tags **NO son retroactivas** (no aplican a recursos creados antes de activarlas)

### Estrategia de tagging
- Crítico como SA: dejar tags bien definidos al armar infra para clientes
- Tags comunes: user:Project, user:Env, user:Team, user:Owner
- Sin tags → no se puede rastrear quién gasta qué
