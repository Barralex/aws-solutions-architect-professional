# 6.3 Amazon S3

## Anti-patterns (NO usar S3 para...)
- Muchos archivos pequeños
- POSIX filesystem (usá EFS)
- File locking
- Búsquedas/queries (usá DynamoDB para indexar)
- Contenido dinámico rápido
- Montar en EC2 como filesystem

## Replication
- CRR = Cross Region Replication
- SRR = Same Region Replication
- Requiere versioning
- RTC (Replication Time Control) = 99.99% en 15 min

## Event Notifications
- Triggers: ObjectCreated, ObjectRemoved, ObjectRestore, Replication
- Destinos básicos: SNS, SQS, Lambda
- EventBridge: filtros avanzados (metadata, size), múltiples destinos, replay

## Performance

### Por prefijo
- 3,500 PUT/COPY/POST/DELETE por segundo
- 5,500 GET/HEAD por segundo
- Más prefijos = más throughput

### Multi-Part Upload
- Recomendado: >100MB
- Obligatorio: >5GB
- Upload en paralelo
- Lifecycle policy para limpiar uploads incompletos

### Transfer Acceleration
- Usa edge locations → red privada AWS
- Para uploads cross-region

### Byte-Range Fetches
- Download en paralelo por rangos
- O descargar solo parte del archivo (ej: header)
