# 10.14 Amazon QuickSight

## Qué es

- **Serverless BI (Business Intelligence)** service
- Dashboards interactivos conectados a data sources
- Fast, auto-scalable, embeddable en websites
- **Per-session pricing**

## Casos de uso

- Business analytics
- Visualizaciones
- Ad-hoc analysis visual
- Business insights desde data

## SPICE Engine

- **In-memory computation engine**
- Solo funciona si importás data DENTRO de QuickSight
- NO funciona si QuickSight está conectado directo a otra DB
- Super rápido por ser in-memory

## Integraciones

### AWS Data Sources:
- RDS, Aurora
- Redshift (data warehouse)
- Athena (ad-hoc queries sobre S3)
- S3 (importar data)
- OpenSearch
- Timestream

### Third-party SaaS:
- Salesforce, Jira, etc.

### Third-party Databases:
- Teradata, on-premises DBs via JDBC

### File Import (usan SPICE):
- Excel, CSV, JSON, TSV, ELF/CLF (logs)

## Users, Groups & Dashboards

### Users & Groups
- **Users**: Standard + Enterprise edition
- **Groups**: Solo Enterprise edition
- Users/Groups existen SOLO dentro de QuickSight (NO son IAM users)
- IAM users = solo para administración

### Dashboard vs Analysis
- **Analysis**: versión completa, editable
- **Dashboard**: snapshot read-only de un analysis
- Dashboard preserva: filtros, parámetros, controles, sort
- Se comparten con users/groups específicos
- Hay que publicar primero para compartir
- Si user tiene acceso al dashboard → ve underlying data

### Column-Level Security (CLS)
- Solo **Enterprise edition**
- Prevenir que ciertos users vean ciertas columnas
