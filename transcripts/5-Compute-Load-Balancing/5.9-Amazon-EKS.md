# 5.9 Amazon EKS

**Curso:** AWS Solutions Architect Professional (SAP-C02)
**Sección:** 5 - Compute & Load Balancing
**Duración:** ~5 min

---

## Resumen

EKS = Kubernetes managed en AWS. Alternativa a ECS pero con API Kubernetes (open-source, cloud-agnostic). Tres tipos de nodes: Managed, Self-managed, Fargate. Storage via CSI driver (EBS, EFS, FSx). Pods = equivalente a ECS Tasks.

---

## ECS vs EKS

| ECS | EKS |
|-----|-----|
| API propietaria de AWS | API Kubernetes (open-source) |
| Solo AWS | Cloud-agnostic (Azure, GCP, etc) |
| Tasks | Pods |

**Cuándo usar EKS:**
- Ya usás Kubernetes on-premise
- Querés migrar entre clouds
- Querés API estándar de Kubernetes

---

## Tipos de Nodes

| Tipo | Quién maneja | Notas |
|------|--------------|-------|
| **Managed Node Groups** | AWS | ASG managed por EKS, On-Demand/Spot |
| **Self-Managed Nodes** | Vos | Más control, vos creás y registrás |
| **Fargate** | Nadie (serverless) | Sin nodes visibles |

## Storage (CSI Driver)

| Storage | Fargate compatible |
|---------|-------------------|
| EBS | ❌ |
| **EFS** | ✅ (único que funciona con Fargate) |
| FSx for Lustre | ❌ |
| FSx for NetApp ONTAP | ❌ |

---

## Puntos Clave Examen

1. **Kubernetes** = open-source, cloud-agnostic
2. **Pods** = equivalente a Tasks en ECS
3. **EFS** = único storage compatible con Fargate en EKS
4. **Migración entre clouds** → EKS (API estándar)
5. **CSI driver** = Container Storage Interface
