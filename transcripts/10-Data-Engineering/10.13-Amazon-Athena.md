# 10.13 Amazon Athena

## Qué es

- **Query engine serverless sobre S3**
- SQL estándar (motor Presto)
- No mueve datos, analiza directo en S3
- Pago: $5 por TB escaneado

## Formatos soportados

- CSV, JSON, ORC, Avro, Parquet

## Uso común

- Athena → QuickSight (dashboards)
- Ad hoc queries, BI, analytics
- Analizar logs: VPC Flow Logs, ALB Logs, CloudTrail

## Performance Tips

### 1. Formato columnar
- Usar **Parquet** o **ORC** (menos data escaneada)
- Glue ETL para convertir CSV → Parquet

### 2. Compresión
- bzip2, gzip, lz4, snappy, zstd

### 3. Partitioning
- Organizar S3 por columnas
- `s3://bucket/flight/year=1991/month=1/day=1/data.parquet`
- Query filtra por partición → escanea menos

### 4. Archivos grandes
- Mínimo 128MB por archivo
- Muchos archivos chicos = overhead

## Federated Query

- Query datos fuera de S3
- Data Source Connector = Lambda function
- Sources: RDS, DynamoDB, Redshift, ElastiCache, DocumentDB, on-premises
- Resultados se guardan en S3
