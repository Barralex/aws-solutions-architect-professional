# 8.4 Aurora - Basics

## Solo para examen (no lo vas a usar)

**Engines:** PostgreSQL y MySQL únicamente

## Números que caen

| Qué | Valor |
|-----|-------|
| Storage auto-grow | Hasta 256 TB |
| Copias de datos | 6 copias en 3 AZs |
| Read replicas | Hasta 15 |
| Failover | < 30 segundos |
| Writes requieren | 4/6 copias |
| Reads requieren | 3/6 copias |

## Endpoints (CAE EN EXAMEN)

| Endpoint | Para qué |
|----------|----------|
| Writer | Writes → Master |
| Reader | Reads → Load balance a replicas |
| Custom | Subset de replicas (ej: analytics en instancias grandes) |
| Instance | Debug/diagnóstico de 1 instancia específica |

## vs RDS

| Feature | RDS | Aurora |
|---------|-----|--------|
| Failover | ~60 seg | <30 seg |
| Read replicas | 5 max | 15 max |
| Storage | Manual/auto-scale | Auto (hasta 256TB) |
| Multi-AZ | Configurable | Por defecto |
| Reader endpoint | NO (usás Route 53) | SÍ (built-in) |

## vs DynamoDB (cross-region)

```
Aurora cross-region  → Copia TODA la DB
DynamoDB Global Tables → Puede ser 1 tabla
```

## S3 Integration

```
Aurora ←→ S3 directo (sin app intermedia)
- Export a S3
- Load desde S3
```
