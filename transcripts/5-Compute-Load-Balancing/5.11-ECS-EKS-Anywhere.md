# 5.11 ECS Anywhere & EKS Anywhere

**Curso:** AWS Solutions Architect Professional (SAP-C02)
**Sección:** 5 - Compute & Load Balancing
**Duración:** ~5 min

---

## Resumen

ECS Anywhere = correr contenedores on-premise manejados desde AWS (requiere conexión). EKS Anywhere = Kubernetes on-premise, puede funcionar sin conexión a AWS. Diferentes modos de conectividad.

---

## ECS Anywhere

### Qué es
Correr ECS Tasks en tu datacenter on-premise, manejados desde la consola de AWS.

### Cómo funciona

```
AWS Cloud                    On-Premises
─────────                    ───────────
ECS Service ◄───internet────► Servidores con:
SSM Service                    • ECS Agent
                               • SSM Agent

Launch type: EXTERNAL
```

### Requisitos
- Instalar ECS Agent + SSM Agent on-premise
- Conexión estable a AWS
- Launch type = **external**

### Casos de uso
- Compliance/regulatorio
- Baja latencia a servicios on-premise
- ML, video processing cerca de los datos

---

## EKS Anywhere

### Qué es
Kubernetes cluster on-premise usando **EKS Distro** (la versión de K8s de Amazon).

### Modos de operación

```
FULLY CONNECTED / PARTIALLY DISCONNECTED
─────────────────────────────────────────
On-Premises EKS ◄──── EKS Connector ────► AWS EKS Console
                      (conexión a AWS)
→ Manejás el cluster desde AWS


FULLY DISCONNECTED
──────────────────
On-Premises EKS (solo EKS Distro)
Sin conexión a AWS
→ Usás herramientas open-source para manejar
```

### Diferencia clave con ECS Anywhere

| ECS Anywhere | EKS Anywhere |
|--------------|--------------|
| REQUIERE conexión a AWS | PUEDE funcionar sin conexión |
| Siempre manejado desde AWS | Puede ser 100% on-premise |

---

## Puntos Clave Examen

1. **ECS Anywhere** = necesita conexión, launch type "external"
2. **EKS Anywhere** = puede ser fully disconnected
3. **EKS Distro** = la versión de Kubernetes de Amazon
4. **EKS Connector** = para conectar EKS on-premise a AWS (opcional)
