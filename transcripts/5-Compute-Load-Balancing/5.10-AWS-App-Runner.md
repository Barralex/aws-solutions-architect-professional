# 5.10 AWS App Runner

**Curso:** AWS Solutions Architect Professional (SAP-C02)
**Sección:** 5 - Compute & Load Balancing
**Duración:** ~4 min

---

## Resumen

App Runner = deploy web apps/APIs sin saber nada de infraestructura. Le das código o imagen Docker, configurás CPU/RAM, y listo. Auto scaling, load balancing, encryption incluidos. Puede conectarse a VPC.

---

## Cómo funciona

```
Tu código / Docker image
         │
         ▼
    App Runner
    (configurás CPU, RAM, auto scaling)
         │
         ▼
    URL pública lista para usar
```

No sabés qué hay debajo. No ves EC2, ECS, nada.

---

## Qué incluye automáticamente

- Auto scaling
- High availability
- Load balancing
- Encryption
- Acceso a VPC (DBs, cache, queues)

---

## Arquitectura Multi-Region

```
┌─────────────────────┐     ┌─────────────────────┐
│     us-east-1       │     │     us-west-2       │
│                     │     │                     │
│  ECR ──────────────────────► ECR (replicado)   │
│   │                 │     │   │                 │
│   ▼                 │     │   ▼                 │
│ App Runner          │     │ App Runner          │
│   │                 │     │   │                 │
│   ▼                 │     │   ▼                 │
│ DynamoDB ──────────────────► DynamoDB (global) │
└─────────────────────┘     └─────────────────────┘
         │                           │
         └───────────┬───────────────┘
                     │
              Route 53 (latency)
                     │
                  Usuario
```

---

## Cuándo usar App Runner

- Deploy rápido de web apps/APIs
- No querés saber de infraestructura
- Microservicios simples
- Startups, prototipos, apps pequeñas

---

## Puntos Clave Examen

1. **Más simple que ECS/EKS** - no ves nada de infra
2. **Source:** código o Docker image
3. **VPC access** - puede conectar a DBs, cache
4. **Multi-region** - ECR replication + DynamoDB Global + Route 53 latency
