# 5.22 AWS Global Accelerator

## Qué es
- Usa la red interna de AWS (no internet público)
- 2 Anycast IPs estáticas → Edge location más cercano → tu app
- Targets: EC2, ALB, NLB, Elastic IPs (públicos o privados)

## Beneficios
- Performance consistente (red privada AWS)
- Fast failover (< 1 min) con health checks
- IPs estáticas (no cambian, fácil whitelist)
- DDoS protection (AWS Shield)
- Soporta TCP y UDP (no solo HTTP)

## CloudFront vs Global Accelerator

| | CloudFront | Global Accelerator |
|---|------------|-------------------|
| Qué hace | Cachea contenido en Edge | Proxea tráfico via red AWS |
| Contenido | HTTP (estático/dinámico) | TCP/UDP (cualquier protocolo) |
| Cache | Sí | No |
| IPs estáticas | No | Sí (2 Anycast) |
| Uso | Web, videos, APIs | Gaming, VoIP, IoT, static IP |

## Para el examen
- Necesito IPs estáticas globales → Global Accelerator
- Gaming/UDP/VoIP → Global Accelerator
- Cachear contenido → CloudFront
- Failover rápido multi-región → Global Accelerator
