# 12.5 SAM - Serverless Application Model

## Conceptos Clave

### Qué es SAM
- **Framework** para desarrollar y deployar aplicaciones serverless
- Todo se define en **YAML**
- Usa **CloudFormation** por debajo (SAM templates se transforman en CF stacks)

### Qué puede crear
- Lambda Functions
- DynamoDB Tables
- API Gateway
- Step Functions State Machines

### Desarrollo Local
- SAM puede correr **Lambda, API Gateway y DynamoDB localmente**
- Facilita un ciclo de desarrollo rápido

### SAM + CodeDeploy (CLAVE PARA EXAMEN)
- Cuando SAM deploya **nuevas versiones de Lambda**, usa **CodeDeploy internamente**
- CodeDeploy hace **traffic shifting** entre versión vieja y nueva (canary/linear)
- Esto es automático, SAM lo invoca por sí solo

### CICD Architecture con SAM

```
CodeCommit → CodeBuild → CodePipeline → CloudFormation/SAM
                                              │
                              ┌────────────────┴────────────┐
                              ▼                             ▼
                        CodeDeploy                    API Gateway /
                    (Lambda traffic shift)            DynamoDB
                     v1 → 10% → 100% → v2         (CF normal)
```

**Pipeline completo:**
1. **CodeCommit** - almacena el código
2. **CodeBuild** - build, test, package
3. **CodePipeline** - orquesta todo
4. **CloudFormation + SAM** - deploy step
5. **CodeDeploy** - traffic shifting para Lambda (automático via SAM)

## Regla para el examen

| Recurso | Cómo lo deploya SAM |
|---|---|
| **Lambda** | Via **CodeDeploy** con traffic shifting |
| **API Gateway, DynamoDB** | Via **CloudFormation** normal |

> **SAM = CloudFormation para serverless + CodeDeploy automático para Lambda**
