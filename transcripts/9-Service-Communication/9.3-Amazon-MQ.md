# 9.3 Amazon MQ

## Qué es

- Message broker **managed** para RabbitMQ y ActiveMQ
- Usa protocolos abiertos: MQTT, AMQP, STOMP, Openwire, WSS
- Para migrar apps on-prem SIN re-escribir código

## Por qué existe

SQS/SNS = protocolos propietarios de AWS
Amazon MQ = protocolos estándar de la industria

## Diferencias con SQS/SNS

| | SQS/SNS | Amazon MQ |
|---|---------|-----------|
| Escalabilidad | Infinita | Limitada (corre en servers) |
| Protocolo | AWS propietario | MQTT, AMQP, STOMP, etc. |
| HA | Built-in | Multi-AZ con failover (manual setup) |
| Caso de uso | Cloud-native | Migración de on-prem |

## Qué incluye

- Queue feature (como SQS)
- Topic feature (como SNS)
- Todo en un solo broker

## Migraciones compatibles

- IBM MQ / WebSphere MQ → Amazon MQ
- TIBCO EMS → Amazon MQ
- RabbitMQ on-prem → Amazon MQ
- Apache ActiveMQ → Amazon MQ

## Para el examen

| Ves en pregunta | Respuesta |
|-----------------|-----------|
| Migrar app on-prem con MQTT/AMQP | Amazon MQ |
| Migrar IBM MQ a AWS | Amazon MQ |
| Cloud-native, máxima escala | SQS/SNS (NO Amazon MQ) |
| Protocols abiertos managed | Amazon MQ |
| RabbitMQ managed en AWS | Amazon MQ |
