# 11.5 - AWS Health Dashboard (Personal Health Dashboard)

## Transcript
AWS Health Dashboard: how AWS outages and maintenance events affect YOUR resources, and how to automate reactions.

## Key Concepts

### What is it
- Bell icon top-right of AWS Console
- Global service
- Shows how AWS outages/maintenance directly impact YOUR resources
- Lists issues + remediation actions
- Keyword for exam: **maintenance events**

### Key Features
- Event Log: history of all events affecting your account
- Dashboard: open issues, scheduled changes, notifications
- **AWS Health API**: access all info programmatically
- **Organizations**: aggregate health events from ALL accounts in one place

### Health Event Notifications (EventBridge Integration)
- EventBridge reacts to health events for YOUR account
- NOT for public Service Health Dashboard events (those use RSS)
- Use cases: notifications, capture event info, automatic corrective actions
- Targets: Lambda, SNS, SQS, Kinesis Data Streams

### Important Distinctions
- **Service Health Dashboard**: public, all AWS services status (RSS feed)
- **Personal Health Dashboard**: YOUR account, YOUR resources (EventBridge)
