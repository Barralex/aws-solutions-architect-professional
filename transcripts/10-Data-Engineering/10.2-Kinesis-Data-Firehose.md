# 10.2 Kinesis Data Firehose

## Qué es

- Servicio para **almacenar streaming data en destinos**
- Fully managed, auto-scaling
- **Near real-time** (no real-time como Streams)

## Arquitectura

```
Sources → Firehose → (Transform Lambda) → Batch Write → Destinations
```

## Sources (de dónde lee)

- Kinesis Data Streams (más común)
- Apps directas (SDK)
- CloudWatch Logs
- AWS IoT

## Destinations (MEMORIZAR)

### AWS:
- **S3** (principal)
- **Redshift** (via S3 + COPY command)
- **OpenSearch**

### Third-party:
- Splunk, Datadog, New Relic, MongoDB

### Custom:
- Cualquier HTTP endpoint

### Backup:
- S3 bucket para datos fallidos o todos los source records

## Buffer (cómo escribe)

- **Buffer size**: ej. 32MB → flush cuando llega a ese tamaño
- **Buffer time**: ej. 2 min → flush aunque no esté lleno
- Mínimo buffer time: **1 minuto**
- Lo que pase primero, dispara el flush

## Transformaciones

- Lambda para transformar (CSV → JSON, etc.)
- Conversión de formato: JSON → Parquet/ORC (para S3)
- Compresión: GZIP, ZIP, SNAPPY (solo Redshift soporta GZIP)

## Manejo de errores

- Transformation failures → S3 backup bucket
- Delivery failures → S3 backup bucket
- Source records → S3 backup bucket (opcional)
- **No pierdes datos nunca**

## Streams vs Firehose

| | Data Streams | Firehose |
|---|--------------|----------|
| Latencia | Real-time (70-200ms) | Near real-time (1+ min) |
| Código | Custom producer/consumer | No code |
| Scaling | Manual (shards) | Automático |
| Storage | 1-365 días, replay | Sin storage |
| Destinos | Apps, Lambda, etc. | S3, Redshift, OpenSearch, Splunk |
| Pricing | Por shard/hora | Por GB procesado |

## IMPORTANTE para examen

- **Spark/KCL NO leen de Firehose** (solo de Streams)
- Redshift: Firehose → S3 → COPY command
- Buffer mínimo: 1 minuto

## Para el examen

| Ves en pregunta | Respuesta |
|-----------------|-----------|
| Streaming a S3/Redshift sin código | Firehose |
| Near real-time ETL | Firehose + Lambda |
| Convertir JSON a Parquet | Firehose |
| Streaming con replay | Data Streams (NO Firehose) |
| Auto-scaling streaming | Firehose |
| Spark consume streaming | Data Streams (NO Firehose) |
