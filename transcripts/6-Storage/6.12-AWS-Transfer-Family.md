# 6.12 AWS Transfer Family

## Qué es
FTP/FTPS/SFTP hacia S3 o EFS (sin usar APIs de S3).

## Protocolos
| Protocolo | Encriptado |
|-----------|------------|
| FTP | NO |
| FTPS | Sí (SSL) |
| SFTP | Sí (SSH) |

## Autenticación
- Interna (dentro del servicio)
- Externa: AD, LDAP, Okta, Cognito, custom

## Endpoint Types (IMPORTANTE para examen)

### 1. Public Endpoint
```
Internet → Transfer Family (IP pública manejada por AWS)
                    │
                    ▼
                 S3 / EFS
```
- IP puede cambiar → usar DNS
- NO podés filtrar por IP origen (no security groups)

### 2. VPC Endpoint - Internal
```
VPC / Corporate (VPN/DX) → Transfer Family (IP privada estática)
                                    │
                                    ▼
                                 S3 / EFS
```
- IPs privadas estáticas
- Security Groups / NACLs ✅
- Solo acceso interno

### 3. VPC Endpoint - Internet-Facing
```
Internet ──────────┐
                   ▼
VPC / Corporate → Transfer Family (Elastic IP - vos la controlás)
                        │
                        ▼
                     S3 / EFS
```
- Elastic IP (estática, tuya)
- Security Groups ✅
- Acceso internet + interno

## Cuándo usar cuál

```
Solo interno (VPN/DX)           → VPC Internal
Internet + control de IPs       → VPC Internet-Facing + Elastic IP
Internet sin restricciones      → Public Endpoint
```

## Para el examen

```
FTP a S3/EFS                    → Transfer Family
Filtrar por IP origen           → VPC Endpoint (NO public)
IP estática desde internet      → VPC Internet-Facing + Elastic IP
Integrar con AD/LDAP            → Transfer Family soporta
```
