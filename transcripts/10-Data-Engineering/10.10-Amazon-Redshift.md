# 10.10 Amazon Redshift

## Qué es

- **Data Warehouse** basado en PostgreSQL
- **OLAP** (analytics), NO OLTP (transacciones)
- Columnar storage (eficiente para SUM, AVG, etc.)
- MPP (Massively Parallel Query Execution)

## Modos

- **Provisioned**: servidores on-demand
- **Serverless**: sin infra

## Cluster

- Leader node: query planning + aggregate results
- Compute nodes: ejecutan queries
- 100+ nodos, 16 TB/nodo
- **Single AZ** (multi-AZ disponible para algunos tipos)
- VPC, IAM, KMS encryption, CloudWatch

## Cargar datos

- S3 → COPY command
- Kinesis Data Firehose (near real-time)
- DynamoDB
- DMS (Database Migration Service)

## Enhanced VPC Routing

- COPY/UNLOAD pasa por VPC (no internet)
- Mejor performance, menor costo

## Snapshots & DR

### Automated snapshots
- Cada 8 horas o cada 5GB de cambios
- Retención configurable (ej. 30 días)

### Manual snapshots
- Se mantienen hasta que los borrás

### Cross-Region Snapshot Copy
- Automático a otra región
- Para KMS encrypted: necesita **Snapshot Copy Grant**
- Restore crea **nuevo cluster** (no sobre el existente)

## Redshift Spectrum

- Query data en S3 SIN cargar en Redshift
- Necesita cluster Redshift activo
- Miles de nodos virtuales Spectrum hacen el query
- Resultado → compute nodes → leader node

## Workload Management (WLM)

- Manejar prioridades de queries
- Queues: superuser, short-running, long-running
- Automatic WLM o Manual WLM
- Evitar que queries largas bloqueen las cortas

## Concurrency Scaling

- Auto-agregar nodos cuando hay muchos usuarios
- Cluster adicional temporal
- Cobro por segundo
- WLM decide qué queries van al scaling cluster

## Para el examen

| Ves en pregunta | Respuesta |
|-----------------|-----------|
| Data warehouse en AWS | Redshift |
| Analytics sobre petabytes | Redshift |
| Columnar storage | Redshift |
| Query S3 sin cargar datos | Redshift Spectrum |
| Cross-region DR para Redshift | Snapshot copy (+ Grant si KMS) |
| Queries esporádicas sobre S3 | Athena (NO Redshift) |
| Prevent short queries blocked | WLM queues |
| Muchos usuarios concurrentes | Concurrency Scaling |
| OLTP | RDS (NO Redshift) |
