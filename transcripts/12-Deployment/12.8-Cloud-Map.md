# 12.8 AWS Cloud Map

## Conceptos Clave

### Qué es Cloud Map
- Servicio de **service discovery** totalmente gestionado
- Permite que los servicios se encuentren entre sí dinámicamente
- Sin hardcodear endpoints ni cambiar código al deployear nuevas versiones

### Cómo funciona
1. Servicios se **registran** en Cloud Map con atributos y ubicación
2. Otros servicios **consultan** Cloud Map para descubrir endpoints
3. Al deployear nueva versión → API call a Cloud Map para actualizar registro
4. Próxima consulta devuelve el nuevo endpoint
5. **Cero cambios de código** en los consumidores

### Características
- **Health checks** integrados (no envía tráfico a endpoints caídos)
- Se consulta via **SDK, API o DNS** (integrado con Route 53)

### Cloud Map vs Load Balancer
- **Load Balancer** = distribuye tráfico entre instancias
- **Cloud Map** = service discovery (directorio de dónde están los servicios)
- Pueden usarse juntos

## Regla para el examen
> "Service discovery" o "servicios que se encuentren dinámicamente" → **Cloud Map**
