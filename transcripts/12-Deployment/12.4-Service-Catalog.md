# 12.4 - Service Catalog

## Resumen
Portal de autoservicio para usuarios que no conocen AWS en profundidad. Los admins crean productos (templates CF) agrupados en portfolios, y los usuarios solo pueden lanzar lo que tienen autorizado.

## Conceptos Clave

### Qué es
- Portal de autoservicio controlado para usuarios nuevos/no-técnicos
- Basado en CloudFormation por debajo
- Governance, compliance y consistencia

### Cómo funciona
1. **Admins** crean Productos (= templates de CloudFormation)
2. Agrupan productos en **Portfolios** (colecciones)
3. Asignan **IAM roles** a cada portfolio (quién puede acceder)
4. **Usuarios** ven solo los productos autorizados
5. Usuarios lanzan productos → CF despliega la infra

### Por qué existe
- Usuarios no necesitan saber AWS en profundidad
- Estandarización de tags, configuración, seguridad
- Menos control al usuario, más control al admin
- Integración con ServiceNow (portal de autoservicio externo)

## Para el Examen
- "Usuarios no-técnicos necesitan lanzar infra" → Service Catalog
- "Estandarizar/gobernar lo que los usuarios pueden crear" → Service Catalog
- Producto = CloudFormation template
- Portfolio = colección de productos con permisos IAM
- Se integra con ServiceNow
