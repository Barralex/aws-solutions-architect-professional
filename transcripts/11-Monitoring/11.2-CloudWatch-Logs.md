# 11.2 - CloudWatch Logs

## Transcript
Deep dive into CloudWatch Logs: sources, structure, exports, subscriptions, and multi-account aggregation.

## Key Concepts

### Getting Data INTO CloudWatch Logs
- SDK, CloudWatch Logs Agent, CloudWatch Unified Agent
- AWS services: Elastic Beanstalk, ECS, Lambda, VPC Flow Logs, API Gateway, CloudTrail, Route 53 DNS queries
- CloudWatch Agent works on EC2 AND on-premise VMs

### Log Structure
- **Log Groups**: represent an application (any name)
- **Log Streams**: instances within the app (containers, files)
- **Expiration**: never expire, or 30 days, etc.
- **Encryption**: KMS supported

### CloudWatch Logs Destinations
- Amazon S3 (export)
- Kinesis Data Streams
- Kinesis Data Firehose → S3, ElasticSearch, Redshift, Splunk
- Lambda → ElasticSearch

### Metric Filters & Logs Insights
- **Metric Filter**: filter expression on logs → creates a metric → triggers CloudWatch Alarm
  - Example: count "ERROR" occurrences, find specific IP
- **Logs Insights**: query logs directly within CloudWatch, add results to dashboards

### S3 Export (Batch - NOT real-time)
- Uses `CreateExportTask` API call
- Takes up to **12 hours** to become available
- Supports SSE-S3 and SSE-KMS encryption
- NOT automated, NOT real-time

### Subscription Filters (Real-time)
- Filter logs and send to destinations in real-time/near-real-time:
  - **Lambda** → ElasticSearch (real-time) or custom processing
  - **Kinesis Data Firehose** → S3, ElasticSearch (near-real-time, ~1 min lag)
  - **Kinesis Data Streams** → Firehose, Analytics, KCL, Lambda

### Multi-Account & Multi-Region Log Aggregation
- Each account/region: CloudWatch Logs → Subscription Filter → Kinesis Data Streams (central account)
- Central account: Kinesis Data Streams → Kinesis Data Firehose → S3
- Common architecture for centralized logging/security

### CloudWatch Agent Installation via SSM
- **SSM Run Command**: `ConfigureAWSPackage` → `AmazonCloudWatchAgent`
- **SSM State Manager**: ensures agent stays installed
- **Manual install**: agent downloads config from SSM Parameter Store
