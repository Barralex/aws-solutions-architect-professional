# 10.8 Running Jobs on AWS - Comparison

## 6 Estrategias

### 1. EC2 + CRON (NO recomendado)
- No HA, no escalable
- Si EC2 muere, job muere

### 2. EventBridge + Lambda (serverless CRON)
- Serverless, HA
- Limitado por Lambda (15 min, runtime)
- AWS lo recomienda siempre que sea posible

### 3. Reactive Workflows (event-driven)
- Lambda invocada por eventos
- Sources: EventBridge, S3, API Gateway, SQS, SNS
- Reaccionar a cambios en infraestructura

### 4. EventBridge + Batch
- Jobs largos, full Docker containers
- Scheduleable (ej: cada 3 horas)
- Más capacidades que Fargate solo

### 5. EventBridge + Fargate
- Más bare-bones que Batch
- Correr un container rápido
- Menos features de scheduling/scaling

### 6. EMR
- Big data jobs (Spark, Hadoop)
- Step execution
- Clusters de big data

## Para el examen

| Ves en pregunta | Respuesta |
|-----------------|-----------|
| CRON serverless | EventBridge + Lambda |
| Job largo sin límite | EventBridge + Batch |
| Reaccionar a eventos | Lambda (event-driven) |
| Big data processing | EMR |
| Container rápido scheduled | EventBridge + Fargate |
| EC2 con CRON | NUNCA recomendar |
