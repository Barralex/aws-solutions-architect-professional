# 6.13 AWS Storage Services Price Comparison

## Ranking de precios (memorizar para examen)

```
MÁS CARO $$$$
─────────────────────────────
1. EBS io1/io2      ← Databases, IOPS provisioned
2. EFS Standard     ← Network FS, multi-AZ
3. EFS One-Zone
4. EBS gp2/gp3      ← General purpose SSD
5. EBS st1          ← HDD, sequential reads
6. EFS IA ≈ S3 Standard
7. EBS sc1          ← Cold HDD (3x más barato que st1)
8. EFS One-Zone IA  ← Cheapest EFS/EBS
9. S3 Standard IA+  ← Cheapest overall
─────────────────────────────
MÁS BARATO $
```

## Reglas simples

```
Más performance = más caro
Más redundancia = más caro
Menos acceso = más barato
```

## Para el examen

```
"Cheapest block storage"        → EBS sc1 (cold HDD)
"Cheapest shared filesystem"    → EFS One-Zone IA
"Cheapest overall"              → S3 (IA, Glacier, etc.)
"Need IOPS, cost doesn't matter"→ EBS io1/io2
```

## Trampas comunes

- EFS IA ≈ S3 Standard en precio
- EFS One-Zone IA < EBS sc1 (pero perdés si cae la AZ)
- gp2/gp3 más caro que st1/sc1 (SSD vs HDD)
